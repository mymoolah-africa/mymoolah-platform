## Summary
Discovered that a real R10.00 Capitec RTP paid at 09:39 on 16 Mar 2026 was never credited to the MyMoolah wallet. Root cause: SBSA batch callbacks use the UETR (UUID format) as `orgnlMsgId`, but the DB only stored `msgId` (MMRTP format) — so the batch callback lookup always failed silently. Fixed by storing the UETR in `requestId` and adding a fallback lookup in `processRtpCallback`. Standard Bank RTP tested end-to-end and confirmed working.

---

## Key Finding — Root Cause of Missing R10

**Two callback routes fire for every RTP:**
1. **Realtime callback** — `{ClientMessageId}` in URL path = our `MMRTP...` msgId → lookup by `originalMessageId` → **always worked ✅**
2. **Batch callback** — `grpHdr.OrgnlMsgId.Id` in body = SBSA's own UUID (different to our UETR) → lookup by `originalMessageId` failed → **"No RTP request found" silently ❌**

The R10 Capitec RTP at 09:39: Capitec debited the account, ACCC batch callback arrived with a UUID that didn't match, wallet never credited.

**The UETR (UUID generated by `buildPain013`) was being generated but discarded** — never stored, never used for matching.

## Fix Applied (`8a5e9cfe`)
**File**: `services/standardbankRtpService.js`

### 1. Capture UETR from `buildPain013`
```javascript
// Before: const { pain013, msgId } = buildPain013({...})
// After:
const { pain013, msgId, uetr } = buildPain013({...})
```

### 2. Store UETR in `requestId` (model comment always said "UETR from Pain.013")
```javascript
// Before: requestId: msgId
// After:
requestId: uetr,           // UETR (UUID) — SBSA uses this in batch callbacks
originalMessageId: msgId,  // MMRTP... — SBSA uses this in realtime callbacks
```

### 3. Fallback lookup in `processRtpCallback`
```javascript
// Primary lookup unchanged — realtime path still works exactly as before
let rtpRequest = await db.StandardBankRtpRequest.findOne({ where: { originalMessageId } });

// Fallback: SBSA batch callbacks may use UETR as orgnlMsgId
if (!rtpRequest && originalMessageId) {
  rtpRequest = await db.StandardBankRtpRequest.findOne({ where: { requestId: originalMessageId } });
  if (rtpRequest) console.log('[RTP-CB] Matched by requestId (UETR) fallback: orgnlMsgId=%s', originalMessageId);
}
```

### 4. Same fix applied to PBAC retry flow
`retryRtpAsPbac` now also stores `retryUetr` in `requestId`.
