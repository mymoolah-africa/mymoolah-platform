## Key Decisions
- **pain.001.001.03 (not .09)**: SBSA's SSVS system validates against v3. The filename `Pain001v3` = version 3. Their working sample confirms the namespace `urn:iso:std:iso:20022:tech:xsd:pain.001.001.03`. Common in SA banking — many institutions still use older ISO 20022 message versions.
- **Kept InstrId in CdtTrfTxInf**: Melanie's annotated spec explicitly includes InstrId. The working sample omits it (minimal example), but it's a valid optional field in v3 and provides payment-level traceability for Pain.002 reconciliation.
- **InstrPrty NORM (not configurable by rail)**: The working sample uses NORM. If SBSA needs HIGH for RTC, that's a separate conversation. EFT/RTC rail is still tracked on DisbursementRun but no longer changes the XML structure.
- **BOL User ID OWN11 via env var**: Added `SBSA_BOL_USER_ID` env var (default: OWN11). Required in InitgPty/OrgId for SBSA to identify our profile.
- **Filename env vars**: `SBSA_COMPANY_CODE` (MYMOOLAH), `SBSA_BOL_USER_ID` (OWN11), `SBSA_FILE_ENV` (TST for test, PRD for production).
