Standard Bank's Rapid Payments Platform clears the payment instantly and credits the MyMoolah Treasury account. Transaction status becomes ACCC.

6
+
Reference resolution depositNotificationService
Extracts the mobile number reference using a priority chain. The reference identifies which wallet to credit.
// Priority chain
1. cdtrAcct.prxy.id // creditor proxy (ShapID ref)
2. dbtrAcct.prxy.id // debtor proxy
3. rmtInf.ustrd // unstructured remittance
4. cdtrAcct.id.othr.id // creditor account
5. dbtrAcct.id.othr.id // debtor account
ResultMobile number (e.g. 0825571055) for wallet lookup
7
+
Dual idempotency check StandardBankTransaction
Two-layer duplicate detection prevents double-crediting: exact transactionId match + cross-channel guard (same ref + amount within 90s window).
Layer 1Exact transactionId match (type = 'deposit')
Layer 2Same ref + amount, completed, within 90s
Cross-chanPAYSHAP-IN-* vs SBSA-SOAP-*
If dupReturns already_processed — no double-credit
8
+
Wallet lookup by mobile number resolveReference()
Normalizes the reference to E.164 format (+27...) and queries the Users table, then finds the associated Wallet. Also checks float accounts for SUP/CLI/SP/RES prefixes.
QueryUser.findOne({ phoneNumber: e164 }) → Wallet.findOne({ userId })
Found{ type: 'wallet', wallet, userId }
Not foundReturns null → parks in suspense (step 11)
✓ Wallet found → Step 9
✗ No match → Step 11
9
+
Credit wallet (row-level lock) Wallet.credit()
Acquires SELECT FOR UPDATE lock on the wallet, credits the balance atomically, and records a StandardBankTransaction — all within a single ACID database transaction.
// ACID transaction
const tx = await sequelize.transaction();
const wallet = await Wallet.findOne({
  where: { walletId },
  lock: Transaction.LOCK.UPDATE,
  transaction: tx
});
await wallet.credit(amount, 'credit', { transaction: tx });
await StandardBankTransaction.create({...}, { transaction: tx });
await tx.commit();
LockSELECT FOR UPDATE prevents concurrent race
On errortransaction.rollback() — zero balance change
10
+
Post double-entry journal ledgerService
Records the deposit in the double-entry ledger: debit bank account (money received), credit client float (wallet balance increase). Every rand accounted for.
DR 1100-01-01 Bank Inflow     R100.00
CR 2100-01-01 Client Float    R100.00
// ref: SBSA-DEP-PAYSHAP-IN-{txId}
PrincipleDebits = Credits — banking-grade double-entry
On failWarning logged; wallet credit stands (ledger catchup)
11
+
Unresolved → Park in suspense parkInSuspense()
If no wallet or float matches the reference, the deposit is parked in the unallocated receipts ledger and an ops alert is triggered for manual resolution.

CR 2600-01-01 Unallocated Receipts R100.00
// Ops resolves manually
